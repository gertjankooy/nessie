# Creditcard Input

## Usage
### Use when
- A user needs to enter their credit card number.
### Don't use when
_Not available in ZeroHeight — to review._

## Anatomy
_Shares the input anatomy (label, help text, field, error message). See [Input](input.md)._

## Configurations
_Not documented as configurable properties in ZeroHeight._

## Placement
- Usually part of a larger composition of form controls for credit card details.

## Behavior
- The keyboard changes to show numbers only on touch devices.
- Supported card types are recognized automatically based on INN ranges.
- The number is formatted automatically based on the detected card type.

## Best practices
- Compose it with the other credit card detail fields rather than using it standalone.

## Content guidelines
- Use the label "Kaartnummer" and help text "De cijfers aan de voorkant van de kaart." instead of the generic defaults.
