# Label

## Usage
### Use when
- Prior to a form element.
### Don't use when
- Providing an example of the data to enter — use [Help text](https://design.ns.nl/4a05a30ad/v/latest/p/0236ea-help-text) or placeholder text instead.

## Anatomy
- **Label text**: a short word or phrase naming the choice or next step.
- **Optional marker** (optional): the word "optional" in parentheses immediately after the label when the field isn't required.

## Configurations
### Optional
Show "optional" in parentheses directly after the label when an input is not required. Only ask for necessary information; mark the rare optional field rather than the many required ones.

## Placement
- Place labels above their input. Top-aligned labels are completed at a higher rate than left-aligned labels.

## Behavior
_Not available in ZeroHeight — to review._

## Best practices
- Don't use placeholder text as a label — it hides context and creates accessibility issues.
- Use help text for additional information; if only some users need it, attach a tooltip beside the label that opens a popover.

## Content guidelines
- Keep it short — one or two words, max three.
- Use a word that gives guidance, preferably an active verb.
- Use: "Maak foto"
- Don't use: "Foto"
- Always start with a capital, and add "optional" when the field isn't obligatory.

## Accessibility
- Keep the label visible when its input receives focus. `[3.3.2]`
- Associate the label with its input field in code so it's announced on focus. `[1.3.1]`
