# Selectable

## Usage
### Use when
- Selecting a group of content.
- Selecting multiple options of content.
- Presenting options to a user in a structured manner.
- Offering a single-select state (like a radio button) or a multi-select state (like a checkbox).
### Don't use when
- The content would redirect to another page — use the [Tile](https://design.ns.nl/4a05a30ad/p/26346e-tiles) instead.
- Combining selectables and tiles in the same row — selectable is built on the form tokens and components.

## Anatomy
- **Card**: each selectable tile; a set is built from two or more cards.

## Configurations
### Selection mode
- **Single select**: behaves like a radio button.
- **Multi select**: behaves like a checkbox.

## Placement
- Lay cards out as a horizontal or vertical set.
- Don't mix selectables and tiles within the same row.

## Behavior
- Default, hover, active, and focus states behave like other form controls.
- Padding baseline is 20px; the radio-button and checkbox selectable types have a fixed 20px padding.

## Best practices
- Use selectable (not tiles) when the goal is selection rather than navigation.
