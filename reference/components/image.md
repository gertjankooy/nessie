# Image

## Usage
### Use when
- Enhancing the experience and expressing the brand's visual language.
- Creating a sense of context.
### Don't use when
- Just filling up empty space — embrace white space.

## Anatomy
- **Image**: the visual, holding its aspect ratio and reserving space before it loads.
- **Caption** (optional): extra context about the image (not the same as alt text).

## Configurations
### Ratio
Landscape (16:9), Square (1:1), or Panorama (3:1).

### Width
- **Default**: positioned within the grid, with a 12px border radius.
- **Full width**: can sit outside the grid, with no border radius.

### Caption
Optional text providing context about the image.

## Placement
- Default images sit within the grid; full-width images can extend beyond it.

## Behavior
- Reserves space for the image before it loads, so the page doesn't jump when it appears.

## Best practices
- Choose photos or illustrations that express a message and reflect the NS brand (see NS Merkplaats).
- On handoff, discuss file type, size, positioning, and alt text with development.

## Content guidelines
- Use captions for context; keep them distinct from alt text.

## Accessibility
- Always provide alt text (content description on Android, accessibility label on iOS) unless the image is purely decorative. `[1.1.1]`
