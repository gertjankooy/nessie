# Help Text

## Usage
### Use when
- Giving users help in context, such as where to find information, how their data is used, or how many options can be selected.
### Don't use when
- The label itself already provides enough information.

## Placement
- Appears directly after the label, before the input.

## Behavior
- Help text should be available when the input is focused.

## Best practices
- Be careful with help text on a tinted background — contrast can drop too low. Either make the label clear enough on its own, or adjust the help text color (for example to the default text color).

## Content guidelines
- Only use help text when there are technical boundaries to communicate (a maximum number of characters, specific allowed signs) that prevent an error, or to guide what information to fill in.
- Keep it short.
- Write in the active voice.

## Accessibility
- Because it appears directly after the label, users of assistive technology receive the extra context before their data is asked. `[3.3.2]`
