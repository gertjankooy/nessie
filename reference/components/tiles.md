# Tiles

## Usage
### Use when
- Grouping related information and/or actions as one card.
- Providing an entry point linked to more information.
### Don't use when
- Grouping all your content — try chunking with white space first.
- The message is feedback or a status update — use a [Message inline](https://design.ns.nl/4a05a30ad/p/774e05-message-inline).
- Grouping a lot of, or complex, content — use a separate page.

Use the Container when the Default prefab doesn't cover the use case.

## Anatomy
- **Header**: a heading and optional metadata.
- **Content**: any kind of content.
- **Footer**: actions.

## Configurations
### Emphasis
- **Border**: preferred for non-interactive tiles.
- **Border + shadow**: on white backgrounds.
- **Shadow**: only on tinted backgrounds.
- **Tinted**: to highlight specific content.

### Density
Default, Dense, or Custom spacing and insets.

## Placement
- Tiles span full width; their height depends on the content inside.
- The container has 16px padding.

## Behavior
- Tile height grows with its content.

## Best practices
- Keep the components used consistent from tile to tile.
- The heading defaults to 2XL.

## Content guidelines
- Keep it short and active; capitalize each part.
- End the description with a period.
- Use an active verb in calls-to-action.
