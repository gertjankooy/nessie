# Sticker

## Usage
### Use when
- Highlighting information connected to a specific context — pricing, a promotional offer, a status, or drawing attention to content within a tile or component.
### Don't use when
- Highlighting text that stands on its own and isn't connected to other information.

## Anatomy
- **Label**: short, scannable text describing the status or extra information.

## Configurations
### Type
- **Brand**: capture immediate attention or reinforce the NS brand.
- **Default**: noticeable within a component without being the primary focus.
- **Subtle**: a neutral, low-attention status or supporting information.
- **Offer**: a deal, discount, or promotional content.
- **Attention**: warn about something important but not critical.
- **Alert**: a message affecting safety, access, or task completion.
- **Info**: helpful, non-urgent context.
- **Highlight**: spotlight something new or noteworthy ("New", "Beta") without brand colors.

### Size
Default or Compact.

### Style
Filled or outlined.

## Placement
- Position the sticker to clearly identify the object it labels; placement within a [Tile](https://design.ns.nl/4a05a30ad/p/26346e-tiles) affects its attention value.

## Behavior
- Each type/size/style preset carries its own attention level — choose deliberately.

## Best practices
- Lean on the established color patterns so users can read importance quickly.
- Keep labels short and scannable, positioned to identify the object they inform.

## Content guidelines
- Use a maximum of two words (e.g. "Most popular").

## Accessibility
- A sticker never stands alone — combine its label with the element it relates to into one accessible description (for example "NS Voordeel, €19.00, Offer, 20% discount"). `[1.3.1]`
- Include the type only when it adds clarity: announce it for Offer, Attention, and Alert; use content only for Brand, Default, Subtle, and Highlight. `[4.1.2]`
